## Инструкция по шагам

1. ### Прочитай и проанализируй план разработки
 - Внимательно изучи весь план.
 - Пойми структуру, назначение компонентов и взаимосвязи.
 - Не пропускай ни одного пункта.

2. ### Создай структуру проекта
 - Используй **инструменты для создания файлов и директорий** (например, `os`, `pathlib` или встроенные средства среды).
 - Создай **все необходимые папки и подпапки**, как указано в плане.
 - Создай **все файлы**, описанные в плане (например, `.py`, `.json`, `.md`, `.env`, и др.).

3. ### Заполни файлы содержимым
 - Для каждого пункта плана **сгенерируй корректный и работающий код**.
 - Раздели код по модулям и файлам согласно плану.
 - Убедись, что **каждый файл содержит содержательный код**.
 - Используй **инструменты записи в файлы** — не выводи код в консоль, а **всегда сохраняй его в соответствующие файлы**.
 - **Весь Python-код обязательно должен быть заключён в блоки:**
   ```python
   # пример кода
   def hello():
       print("Hello, world!")
   ```
 - **Важно:** В коде **не должно быть символов `\n` внутри строк.**
   Все строки должны быть отформатированы с **настоящими переносами строк**.
   Код должен быть готов для прямой вставки в файл без дополнительной обработки.

4. ### Создай вспомогательные файлы
 - `README.md` — с описанием проекта, структуры, целей, инструкцией по запуску и примерами.
   - **Только форматированный Markdown-текст** (заголовки, списки, параграфы, блоки кода и т. д.).
   - **Не должно быть `\n` в строках. Используй настоящие переводы строк.**
 - `requirements.txt` — со списком всех зависимостей.

## Обязательные технические требования

- Обязательно используй **инструменты для создания и записи файлов**.
- **Никакой код не должен выводиться в текстовом виде** — он должен быть записан в соответствующие файлы.
- **Весь Python-код должен быть оформлен строго внутри блоков ` ```python ... ``` `.**
- **Всё содержимое должно быть с настоящими переводами строк, без `\n` внутри текста.**
- `README.md` должен содержать **только валидный Markdown-текст**, структурированный и читаемый.
- Структура проекта и реализация должны соответствовать уровню **Senior-разработчика**.
